# Dependabot Configuration Changes Summary

## 変更内容 (Changes Made)

Dependabotの設定を変更し、Terraformプロバイダーのアップデートをプロバイダーごとにグルーピングするように設定しました。

### 変更前 (Before)
- 各ディレクトリごとに個別のプルリクエストが作成されていました
- `directories: ["**/*"]` の設定により、30以上のディレクトリそれぞれに対してPRが作成される可能性がありました

### 変更後 (After)
- プロバイダーごとにプルリクエストがグルーピングされるようになりました
- 以下の4つのグループに分類されます：

#### 1. aws-provider グループ
- `registry.terraform.io/hashicorp/aws` プロバイダーのアップデートをまとめます
- 最も使用頻度の高いプロバイダーです

#### 2. oci-provider グループ  
- `registry.terraform.io/oracle/oci` プロバイダーのアップデートをまとめます
- Oracle Cloud Infrastructure関連のプロジェクト用です

#### 3. hashicorp-providers グループ
- 以下のHashiCorp製プロバイダーをまとめます：
  - http
  - null
  - random
  - local
  - tls
  - time

#### 4. other-providers グループ
- 上記以外のすべてのプロバイダーをまとめます
- 将来的に新しいプロバイダーが追加された場合のフォールバック用です

## 効果 (Benefits)

1. **マージ作業の軽減**: 30以上の個別PRから最大4つのグループPRに削減
2. **論理的なグルーピング**: 関連するプロバイダーアップデートが一つのPRにまとまります
3. **レビューの効率化**: 同じプロバイダーの変更を一度に確認できます
4. **既存設定の保持**: 週次スケジュール、メジャーバージョン除外などの既存設定はそのまま維持

## 設定詳細 (Configuration Details)

- **監視対象**: 全ディレクトリ (`directories: ["**/*"]`) - 変更なし
- **スケジュール**: 週次 (`interval: "weekly"`) - 変更なし  
- **対象ブランチ**: main - 変更なし
- **PR制限**: 1000 - 変更なし
- **除外設定**: メジャーバージョンアップデートを除外 - 変更なし
- **新規追加**: `groups` セクションによるプロバイダーグルーピング

この変更により、Dependabotが作成するプルリクエストの数が大幅に削減され、メンテナンスの負荷が軽減されます。
